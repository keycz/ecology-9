数据库表名 workflow_dimension 中文名称 流程待办维度表
所属模块 工作流程 主键 无
说明 用于控制前台待办列表维度展示，同时也为后台流程中心元素设置返回一些数据。
序号 数据库列名 中文名称 数据类型 长度 是否允许空值 是否为外键 是否自增长 默认值 说明
1 id id integer 否 否 是 id，暂时没用，此字段不是主键。
2 typeid 待办维度id integer 否 否 否 字段有唯一性约束，不能为空。
3 typename 待办维度类型名称 varchar2 20 是 否 否 待办维度类型名称，如"flowAll"
4 typetitle 待办维度显示名称 integer 是 否 否 这个是用于返回给前台显示名称，存的是多语言id
5 showcount 维度是否显示数量 char 1 是 否 否 1：显示，其他：不显示
6 groupcolor 维度数量颜色 varchar2 20 是 否 否 如"#000000"
7 sqlwhere 维度对应的查询条件 varchar2 1000 是 否 否 用括号包裹，括号内写入查询需要加的条件，不加则查待办的全部，暂时只支持使用workflow_currentoperator表中的字段做为条件。使用时需使用t2作为别名，如：(t2.workflowid not in(1))
8 typesettingjstr 流程中心元素查看类型辅助选项设置 varchar2 2000 是 否 否 这个是json类型，存放进去的必须为json格式，否则会解析错误。如果设置的话，则必须二开增加相应的辅助项设置，因为这个相当于辅助项，不加这个则无需二开。
9 keyid 流程中心元素查看类型id integer 是 否 否 流程中心元素查看类型id,加了唯一性约束。
10 showcountjstr 流程中心元素显示数量设置 varchar2 2000 是 否 否 这个是json类型，存放进去的必须为json格式，否则会解析错误。
11 url url链接 varchar2 500 是 否 否 用于存放一个url，可以根据url直接访问待办列表指定维度数据
12 dsporder 维度内tab排序 number 是 否 否 维度内tab的排序
13 isshow 是否显示 char 1 是 否 否 1表示默认显示，0表示不显示
14 scope 维度范围 varchar2 10 是 否 否 维度范围，现在有doing（待办）、done（已办）、mine（我的请求）、portal（门户流程中心元素的查看类型）四个范围可用
15 viewcondition 查看参数，给代码使用 number 是 否 否 查看参数，给代码使用，待办维度要大于9，已办维度要大于5，我的请求维度要大于4，同一维度不允许重复。
16 customclass 自定义接口实现类路径 varchar2 100 是 否 否 可以将实现DimensionCustomService接口的实现类放在里面，这个接口有三个方法，实现对应的方法可以拼接对应的sql条件。
17 dataurl 移动端tab数据来源接口 varchar2 500 是 否 否 移动端用于指定对应tab的数据来源接口
18 treeurl 移动端tab左侧树数据来源接口 varchar2 500 是 否 否 移动端用于指定对应tab的左侧树数据来源接口
19 path 移动端tab标识 varchar2 100 是 否 否 前端使用，建议使用：/center/(tab的typename)
20 mobileappkeyid 移动端tab对应的key integer 是 否 否 前端参数，建议和typeid一致即可
21 isdeftab 是否是默认显示的tab char 否 否 否 1默认显示在对应列表的tab中，为0时即使isshow等于1也不会显示
